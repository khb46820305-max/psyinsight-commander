# 경제 흐름 파악 기능 개선안

## 📊 현재 문제점

1. **원래 목적**: "경제 흐름을 한눈에 파악" → **현재**: 단순 뉴스 나열
2. **보고서 생성 실패**: AI 모델 호출 실패, 데이터 부족
3. **사용성 부족**: "흐름" 파악 불가, 중요도 표시 없음, 시간 추적 불가

---

## 🎯 개선안: "스마트 대시보드"

### 핵심 아이디어
**AI 없이도 경제 흐름을 파악할 수 있는 자동 분석 대시보드**

---

## 📋 구체적 개선안

### 1. **카테고리별 요약 카드** (상단)
- 각 카테고리별 뉴스 개수 표시
- 주요 키워드 자동 추출 (빈도수 기반, AI 불필요)
- 최근 3일간 트렌드 (증가/감소 화살표)

### 2. **주요 이슈 하이라이트** (중앙)
- 키워드 빈도수 기반으로 자동 선별
- 제목에 중요 키워드가 많이 포함된 뉴스 우선 표시
- 최대 5개 이슈만 하이라이트

### 3. **날짜별 트렌드 그래프** (하단)
- 최근 7일간 카테고리별 뉴스 개수 변화
- Streamlit의 간단한 차트로 시각화
- 증가/감소 추세를 한눈에 파악

### 4. **키워드 클라우드** (선택)
- 가장 많이 언급된 키워드 크기로 표시
- 카테고리별로 색상 구분

### 5. **개선된 헤드라인 표** (기존 유지)
- 중요도 순으로 정렬 옵션 추가
- 키워드 필터링 강화

---

## 💡 구현 방법

### AI 없이 구현 가능한 기능들
1. **키워드 추출**: 기존 `extract_keywords` 함수 활용 (이미 저장된 키워드 사용)
2. **빈도수 계산**: SQL 집계 함수로 카테고리별 개수, 키워드 빈도 계산
3. **트렌드 분석**: 날짜별 그룹화하여 전일 대비 증가/감소 계산
4. **중요도 점수**: 제목 길이, 키워드 매칭 수, 소스 신뢰도 등으로 계산

### 필요한 데이터
- 이미 수집된 뉴스 데이터 활용
- `keywords` 컬럼의 JSON 데이터 파싱
- 날짜별, 카테고리별 집계

---

## ✅ 기대 효과

1. **"흐름" 파악 가능**: 트렌드 그래프와 카테고리별 요약으로 한눈에 파악
2. **AI 의존성 제거**: 보고서 생성 실패 문제 해결
3. **사용자 편의성 향상**: 중요한 이슈가 자동으로 하이라이트
4. **빠른 로딩**: AI 호출 없이 데이터베이스 쿼리만으로 처리

---

## 🚀 구현 우선순위

1. **1단계**: 카테고리별 요약 카드 + 주요 이슈 하이라이트
2. **2단계**: 날짜별 트렌드 그래프
3. **3단계**: 키워드 클라우드 (선택)

---

이 개선안으로 "경제 흐름 파악"이라는 원래 목적을 달성하면서, AI 의존성 없이 안정적으로 작동하는 기능을 제공할 수 있습니다.
