## Streamlit Cloud 배포 방법

### 1. GitHub 저장소 생성
1. GitHub에 로그인
2. 새 저장소 생성 (예: `psyinsight-commander`)
3. 저장소 URL 복사

### 2. GitHub에 코드 푸시
```bash
# 저장소 연결
git remote add origin https://github.com/사용자명/psyinsight-commander.git

# Personal Access Token 생성 (Settings > Developer settings > Personal access tokens)
# 권한: repo (전체)

# 푸시
git push origin main
```

### 3. Streamlit Cloud 배포
1. https://streamlit.io/cloud 접속
2. GitHub 계정으로 로그인
3. "New app" 클릭
4. 저장소 선택
5. Main file path: `app.py`
6. Secrets에 API 키 추가:
   ```
   GEMINI_API_KEY=your_api_key
   EMAIL_SENDER=your-email@gmail.com
   EMAIL_PASSWORD=your_app_password
   EMAIL_RECEIVER=receiver@example.com
   ```
7. "Deploy" 클릭

### 4. 배포 완료
- 배포 후 자동으로 URL 생성
- 코드 변경 시 자동 재배포

## Streamlit Cloud 앱 비활성화/삭제 방법

### 작업 결과물은 보존하고 웹앱만 중단하고 싶을 때

#### 방법 1: 앱 삭제 (권장)
1. https://streamlit.io/cloud 접속
2. GitHub 계정으로 로그인
3. "Your apps" 메뉴 클릭
4. 삭제할 앱 선택
5. 앱 설정에서 **"Delete app"** 또는 **"Remove"** 버튼 클릭
6. 확인 메시지에서 삭제 확인

**결과:**
- ✅ 코드는 GitHub에 그대로 보존됨
- ✅ 웹앱은 더 이상 접근 불가 (URL 작동 안 함)
- ✅ 나중에 다시 배포 가능

#### 방법 2: 앱 일시 중지 (있다면)
- 일부 Streamlit Cloud 플랜에서는 앱을 일시 중지할 수 있음
- 앱 설정에서 "Pause" 또는 "Stop" 옵션 확인

## 중요 사항

### 코드는 안전합니다
- Streamlit Cloud에서 앱을 삭제해도 **GitHub 저장소의 코드는 그대로 보존**됩니다
- 코드는 `https://github.com/khb46820305-max/psyinsight-commander.git`에 계속 남아있습니다
- 언제든지 다시 배포할 수 있습니다

### 다시 배포하고 싶을 때
- 위의 "Streamlit Cloud 배포 방법" 3단계부터 다시 진행하면 됩니다
- GitHub 코드는 이미 있으므로 바로 배포 가능

## GitHub 인증 방법

### Personal Access Token 생성
1. GitHub Settings > Developer settings > Personal access tokens > Tokens (classic)
2. "Generate new token" 클릭
3. 권한 선택: `repo` (전체)
4. 토큰 생성 후 복사 (한 번만 표시됨)

### 토큰 사용
```bash
git remote set-url origin https://토큰@github.com/사용자명/저장소명.git
git push origin main
```
