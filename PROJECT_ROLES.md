## 🔄 작업 흐름 (Workflow)

### 일반적인 작업 흐름

```
1. [사용자] 자연어 명령 입력
   "Tab 1을 구현해줘"
   ↓
2. [Architect] 작업 분석 및 계획
   - PRD 확인
   - TODO 리스트 확인
   - 구체적인 작업 지시 작성
   ↓
3. [Builder] 실제 구현
   - 코드 작성
   - 모듈별 테스트
   - 결과물 제출
   ↓
4. [QA] 검증 및 리뷰
   - 기능 테스트
   - 코드 리뷰
   - 버그 리포트
   ↓
5. [Architect] 통합 및 최종 확인
   - 모듈 통합
   - todo.md 업데이트
   - 사용자에게 보고
   ↓
6. [사용자] 결과 확인 및 피드백
```

### 현재 단계별 역할

#### ✅ 완료된 단계
1. **Planner**: PRD 및 분석 문서 작성 완료
2. **Architect**: 작업 계획 및 역할 분배 완료

#### 🔄 현재 진행 중
3. **Architect**: 프로젝트 구조 설계 및 초기 설정 준비

#### ⏳ 다음 단계
4. **Builder**: 실제 코드 구현 시작
5. **QA**: 각 Iteration 완료 후 검증
